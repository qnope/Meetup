Small Buffer Optimization (SBO)

Without SBO
Every Drawable allocates on the heap via make_unique.

With SBO
Small objects are stored inline in a fixed-size buffer, avoiding allocation.

std::function and std::any both use SBO internally. 


// Without SBO
std::unique_ptr<Concept> self;
// heap alloc every time
                

// With SBO
alignas(...) char buf[N];
Concept* self;
// placement new if sizeof(T) <= N
// heap alloc otherwise

We won't implement it here — just know it exists and matters for performance.
