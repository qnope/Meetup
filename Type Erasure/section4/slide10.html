Constraining with Concepts (C++20)

Without constraints, passing a wrong type produces unreadable errors deep inside Model<T>. 


// Define the concept
template <typename T>
concept HasDraw = requires(const T& t) {
    t.draw();
};
            

// Constrain the constructor
template <HasDraw T>
Drawable(T obj)
    : self(std::make_unique<Model<T>>(
        std::move(obj))) {}
            

struct NotDrawable { int x; };
Drawable d{NotDrawable{}};

Before: Wall of errors inside Model<T>::draw()
"no member named 'draw' in 'NotDrawable'" — buried in template instantiation.

After: Clear, immediate message
"constraints not satisfied for 'Drawable(T)' [with T = NotDrawable]"

Concepts make the interface explicit and errors actionable.
